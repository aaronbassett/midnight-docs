# Contract upgradability

A deployed Midnight contract is tied to the proof system that compiled it. When that proof system evolves, a contract's circuits can stop verifying, and a contract whose proofs no longer verify is a contract nobody can call. A Contract Maintenance Authority is how you keep the contract alive: it can replace a circuit's verifier key in place, without moving the contract's address, state, or balance.

This tutorial deploys an upgradable counter with an explicit maintenance authority, swaps the verifier key of a live circuit so the same call produces different behavior, then rotates the authority to a fresh key. Every step runs against a real network with a funded wallet.
